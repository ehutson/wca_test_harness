XML API Developer Guide

XML API Test Harness for Recipient and User Actions
Use this application to help build familiarity with Silverpop's XML API.  The intended purpose is to allow you to see the XML of your requests and the subsequent responses, and allow easy experiments with the API.  It is not intended to be a reference implementation of the APIs.

To Use the XML API Test Harness

1. Login
Some APIs require the user to be authenticated and a valid session established.  This is done by selecting and executing the Login API, editing the XML to reflect your user account, and executing it.
If your login is successful, copy the jsession id from the response XML into the jsession field in the left frame.

2. Select/Execute an API
To test one of the APIs, first choose the system Pod, then select the API from the API drop-down.  This will populate the Host and Servlet fields for you.  It will also populate the XML field with boiler plate XML for that API.
You will probably need to edit the XML so that it reflects the proper e-mail address, list id, mailing id, or recipient id that you are testing with.
When your XML is correct, press the submit xml request button.

3. View XML Responses
XML responses will be displayed in the lower right frame (the same one you're reading from right now).  Some browsers will display the raw XML response while others will attempt to render it. If you do not see the XML response, then right click in the response frame and select View Source.  You can cut & paste from here if needed.
